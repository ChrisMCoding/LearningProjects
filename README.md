What is this repo?

This repo is a collection of annotated programs to teach, demonstrate, and explain how to program in python and to help explain principles of programming and best practices. It's designed to take prospective programmers and software developers from having never seen a line of code to writing production-quality code that's maintainable and easy to expand upon.

Neat-o! But there's so much stuff here! How do I use it?

Great question! So there are a few levels of organisation here, first of all you'll see a 'Beginner_A' folder. That's for the very first beginner projects for absolute beginners, specifically those who are just beginning. In the future, there will be 'Beginner_B' and 'Beginner_C' folders, and then similar folders for intermediate and advanced programs.

Inside these folders you'll see numbered folders with names after them. The numbers just tell you what order to look at the programs in and the names are the names of the programs, so '1_Hello_World' is just program 1 and it's called 'Hello World'. Simple so far!

The next layer is a bit more complicated though; why are there so many versions!? So for each program, the first version is the most straightforward way to write the program that I could think of, and each subsequent version does one of the following things:
	A. Adds a new bit of functionality to the program
	B. Reformats some part of the code and explains why it was changed
	C. Demonstrates another way to write the program, or part of the program, and explains anything new in this rewrite
	
The important thing is that you absolutely categorically do not at all need to get through every version of any program before you move onto the next program. If you get to some version that doesn't quite make sense to you and it's just *not* going in, don't worry about it! Just move onto the next program and you can come back to that other version later.

The only recommended way to navigate this repo is just to look at the programs in the order that they're presented. Start with 'Beginner_A', go through programs 1, 2, 3... and so on, and just go up the versions until you're comfortable.

That sounds swell, but there are already so many resources for learning to program, why'd you make this?

Great question! When I was learning to program I made lots of mistakes and I ended up having to unlearn a lot of what I learned when I was starting out. Nowadays, I think that a lot of resources are actually really good and can get you to a really good place on your programming journey. That said, a lot of them are booooooring and require you to like, listen? Or read? And pay attention?

I wanted to write a little guide that you can muddle about in at your own pace, and that's written in a bit of a less professional voice than usual, so it isn't stunningly dull to read. The aim is to make this an engaging and useful tool for anyone starting out with programming. As well as that, I think the best way to learn programming is by doing it, and nothing's better to get you doing stuff than to get you muddling about in an existing program. So this repo provides you with completed programs that you can muddle about in, to try to make learning a little more interesting.

I hope this repo is a bit of fun, and helps you to learn a bit more about how to program!
